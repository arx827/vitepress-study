# React Compiler

## 介紹
  學習 [React 編譯器的作用](/pages/f2e/docReact/reactDev/04/reactDev_04_01.html) 以及它如何透過自動處理記憶化（`memoization`）來優化你的 `React` 應用，從而無需再手動使用 `useMemo`，`useCallback`，和 `React.memo`。

## 安裝
  開始使用與 [安裝 React 編譯器](/pages/f2e/docReact/reactDev/04/reactDev_04_02.html) 並學習如何配置你的建構工具以集成它。

## 漸進式採用
  如果你尚未準備好在整個項目中啟用，學習 [漸進式採用 React 編編譯器的策略](/pages/f2e/docReact/reactDev/04/reactDev_04_03.html)，在現有代碼庫中逐步引入它。

## 調試與故障排除
  當程序未按預期工作時，使用我們的 [調試指南](/pages/f2e/docReact/reactDev/04/reactDev_04_04.html) 以理解編譯器錯誤和運行時問題的區別，識別常見的破壞性模式，並遵循系統化的調試流程。

## 配置與參考
  有關詳細的配置選項和 API 參考：
  - [配置選項](https://zh-hans.react.dev/reference/react-compiler/configuration) - 包含所有編譯器配置選項，包括 React 版本兼容性
  - [指令](https://zh-hans.react.dev/reference/react-compiler/directives) - 用於函數級別的編譯控制
  - [編譯庫](https://zh-hans.react.dev/reference/react-compiler/directives) - 提供預編譯的庫

## 額外的資源
  除了本文檔之外，我們建議你關注 [React 編譯器工作組](https://github.com/reactwg/react-compiler)，以獲取有關編譯器的更多信息和討論。
